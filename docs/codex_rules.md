# Reglas obligatorias para Codex

1. Trabajar siempre en español formal.
2. Respetar flujo de ramas: `modXXXX` siempre desde `develop`.
3. Utilizar commits con formato: `modXXXX - Se ...` y cuerpo con secciones A/B/C/D.
4. No mezclar tareas: cada rama `modXXXX` debe contener un cambio acotado.
5. No introducir dependencias no aprobadas.
6. Mantener base de datos y configuración junto al ejecutable (`AppContext.BaseDirectory`).
7. Mantener logs en `./logs`.
8. Al finalizar cada tarea, listar archivos tocados y detallar cómo probar.
